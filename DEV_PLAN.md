# 心语桥（HeartBridge）— AI 编程开发计划

> **版本**：v1.0
> **工具**：TRAE SOLO / Cursor / Windsurf 等 AI 编程工具
> **日期**：2026-04-29
> **预计工时**：7 天（每天 2-4 小时 AI 对话）

---

## 项目总览

| 项目 | 说明 |
|------|------|
| 产品 | 心智障碍群体职场沟通辅助器 |

## Step 1：项目初始化与首页

**目标**：搭建项目骨架、首页 6 大分类入口

### Prompt 1 — 创建项目结构

```
创建一个移动端优先的单页 Web 应用项目，项目名为"心语桥"（HeartBridge），
用于帮助心智障碍群体在职场中表达需求和情绪。

技术要求：
- 原生 HTML5 / CSS3 / JavaScript，不使用任何框架
- 使用 hash 路由实现 SPA 页面切换
- 所有代码放在一个 index.html 文件中
- 移动端优先，viewport 设置为 width=device-width

设计要求：
- 背景色：暖白 #FFFAF5
- 主色调：温暖珊瑚 #FF8A80
- 强调色：柔和蓝 #A8D8EA
- 成功色：薄荷绿 #95E1D3
- 文字色：深棕灰 #4A4A4A
- 卡片圆角：16px，按钮圆角：24px（胶囊形）
- 页面内边距：20px

请创建完整的 HTML 骨架，包含：
1. 顶部标题栏（"心语桥" + 副标题"职场沟通助手"）
2. 主内容区域（id="app"）
3. 底部导航栏（首页、沟通模板、我的收藏、设置）
4. CSS 变量定义所有设计 token
5. hash 路由系统
```

### Prompt 2 — 首页分类入口

```
在首页实现 6 大表达分类入口网格。

布局：2 列 3 行网格，每个按钮至少 120px x 100px，圆角 20px

6 个分类按钮：
1. 🆘 需要帮助 | 背景色 #FFF0EE，边框 #FF8A80
2. 🤒 身体不舒服 | 背景色 #FFF5E6，边框 #FFB347
3. 💼 工作相关 | 背景色 #E8F8F5，边框 #95E1D3
4. 🏠 请假休息 | 背景色 #E8F8F0，边框 #7EC8A0
5. 👋 社交互动 | 背景色 #EBF5FB，边框 #A8D8EA
6. 😰 我感觉... | 背景色 #F3E8FF，边框 #C9A0DC

每个按钮：
- 大 emoji 居中（40px）
- 分类名称（18px Medium）
- 点击时有按压缩放效果（scale 0.95）
- 点击跳转到 #/category/:id（"我感觉..."跳转到 #/emotion）

底部区域：
- "最近使用"区域：展示最近 3 条使用记录（初始为空状态提示）
- 底部 tab 导航
```

### 验证点

- [ ] 首页 6 个大按钮正确展示
- [ ] 每个按钮颜色和图标正确
- [ ] 点击跳转到对应路由

**目标**：实现情绪选择→推荐表达→管理者建议的完整流程

```

页面一：情绪选择页（#/emotion）

- 引导文字："请选择最接近你现在的感觉："

  😰 紧张 | 背景 #FFF5E6，边框 #FFB347
  😕 困惑 | 背景 #EBF5FB，边框 #A8D8EA
  😢 难过 | 背景 #F3E8FF，边框 #C9A0DC
  😊 开心 | 背景 #E8F8F0，边框 #95E1D3
  😴 累了  | 背景 #F0F0F5，边框 #B0B0C0
- 每个按钮：大 emoji（48px）+ 文字标签（18px）
- 最小尺寸 140px x 100px

页面二：情绪结果页（#/emotion/:id）
- 顶部返回按钮 + 选中情绪的 emoji 和标签
- 两个区域（用不同颜色区分）：

  区域一："你可以这样说"（薄荷绿背景 #E8F8F0）

  - 点击朗读后，按钮短暂变为"已朗读"

  区域二："给管理者的建议"（浅蓝背景 #EBF5FB）

情绪映射数据：
anxious(😰紧张): 
  表达: ["我有点紧张，可以慢慢来吗？", "这个任务让我有点不安，能陪我一起做吗？"]
  建议: 分析"员工可能感到压力过大"，建议: 分解任务/正面鼓励/允许休息/避免催促

  表达: ["我不太明白，能再说一遍吗？", "可以用图片告诉我怎么做吗？"]

sad(😢难过):

happy(😊开心):
  表达: ["我今天很开心！", "我完成了任务，很有成就感！"]
  建议: 分析"员工状态良好"，建议: 及时表扬/增加挑战/鼓励社交/记录表现

tired(😴疲惫):
  表达: ["我有点累了，能休息一下吗？", "我需要喝口水。"]
  建议: 分析"员工可能体力不支"，建议: 安排休息/检查强度/提供饮水/调整节奏
```

## Step 4：沟通模板库

**目标**：实现填空式沟通模板，用下拉选择代替打字

### Prompt 5 — 沟通模板

```
实现沟通模板功能。

模板列表页（#/templates）：
- 顶部标题"沟通模板"
- 4 个模板卡片：

模板一 📋 汇报工作完成
  字段1: 任务名称 → 下拉 [理货, 清洁, 准备材料, 收拾桌面, 其他]

  字段1: 请假类型 → 下拉 [病假, 事假, 半天假]
  字段2: 原因 → 下拉 [身体不舒服, 家里有事, 去医院, 其他]

模板三 🙋 请求帮助
  字段1: 帮助类型 → 下拉 [不明白任务, 找不到工具, 做错了, 需要人帮忙]

- 顶部返回按钮 + 模板名称
- 每个字段是一个大号下拉选择器（高度 56px，圆角 12px，字号 18px）
```

## Step 6：UI 打磨与 PWA 配置

**目标**：动画优化、响应式适配、PWA 离线支持

### Prompt 7 — UI 打磨

```
对心语桥应用进行 UI 打磨：

1. 页面切换：淡入淡出（300ms opacity + translateY）
2. 表达卡片展开：max-height transition（300ms ease）
3. 按钮点击效果：scale(0.95) → scale(1)，100ms
4. 复制成功提示：底部 toast 弹出，2秒后淡出
5. 收藏动画：点击 ⭐ 时缩放弹跳效果
6. 情绪按钮选中态：边框加粗 + 轻微放大
7. 空状态插图：用 CSS 绘制简单的插图（圆形 + emoji）
8. 尊重 prefers-reduced-motion
9. 确保 375px - 768px 响应式正常
```

### Prompt 8 — PWA 配置

```
为心语桥添加 PWA 支持：

1. 创建 manifest.json：
   - name: "心语桥 - 职场沟通助手"
   - short_name: "心语桥"
   - theme_color: #FF8A80
   - background_color: #FFFAF5
   - display: standalone
   - icons: 用 SVG 内联生成一个简单的图标（珊瑚色圆形+白色爱心）

2. 创建 Service Worker（sw.js）：
   - 缓存所有静态资源（Cache First 策略）
   - 离线时返回缓存内容
   - 版本号更新时清理旧缓存

3. 在 index.html 中注册 Service Worker 和 manifest
```

- [ ] 可添加到手机主屏幕
- [ ] 断网后核心功能可用
- [ ] 响应式在 375px 和 768px 下正常

## 注意事项

1. **每个 Prompt 独立执行**：不要在一个 Prompt 中塞太多需求，逐步迭代
2. **及时验证**：每个 Step 完成后都要在浏览器中测试验证
3. **保留中间版本**：用 Git 管理版本，每完成一个 Step 提交一次
4. **截图留存**：开发过程中的关键截图（Prompt 对话、页面效果）留作参赛材料
5. **移动端测试**：使用浏览器开发者工具的移动模拟器测试（375px / 414px）
6. **数据完整性**：28 条表达 + 5 种情绪映射是核心数据，务必逐条验证
