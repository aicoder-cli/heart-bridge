## 一、项目概述

### 1.1 产品名称

**心语桥（HeartBridge）** — 心智障碍群体职场沟通辅助器

### 1.2 一句话描述

帮助心智障碍群体在职场中更顺畅地表达需求和情绪，将"想说但说不出口"的模糊感受转化为清晰语句，减少因沟通障碍导致的误解与冲突。

### 1.3 背景与动机

心智障碍群体（孤独症谱系、智力发育迟缓、唐氏综合征等）在职场中面临的核心挑战之一是**沟通表达困难**。他们常因语言表达能力有限，无法准确传达自己的需求、情绪和想法，导致：

- **被误解**：想说"我不太明白"却说不清楚，被误认为"不听话"或"态度不好"
- **需求被忽视**：身体不舒服、需要帮助时无法及时表达
- **情绪失控**：长期无法表达导致沮丧积累，最终以不当行为爆发
- **社交孤立**：无法与同事正常交流，难以融入团队

现有解决方案（辅导员代为传达、手语卡等）存在覆盖场景有限、依赖辅导员在场、不够灵活等问题。

"心语桥"旨在用技术手段为心智障碍员工提供一个**随身携带的"表达助手"**，让他们能够独立、清晰地表达自己，增强职场融入感和自信心。

### 1.4 目标用户

| 用户角色 | 说明 | 使用场景 |
|----------|------|----------|
| **心智障碍员工（主要用户）** | 孤独症谱系、智力发育迟缓、唐氏综合征等 | 日常工作中表达需求、情绪、汇报 |
| **职场辅导员/管理者（辅助用户）** | 负责监督和支持心智障碍员工 | 查看沟通记录、获取沟通建议、自定义表达内容 |

### 1.5 核心价值

- **让表达变简单**：大图标 + 选择式交互，无需打字，点击即可表达
- **情绪可视化**：将模糊的情绪感受映射为清晰的语句和管理者建议
- **语音辅助**：一键朗读，降低阅读依赖，也用于面对面沟通
- **减少误解**：标准化的表达模板，让沟通更准确、更高效
- **建立自信**：成功表达带来的正向反馈，增强职场社交信心

## 二、产品形态与技术方案

### 2.1 产品形态

- **移动端优先的 Web 应用（PWA）**
- 支持手机/平板浏览器访问
- 支持添加到主屏幕，离线使用
- 无需安装 App，降低使用门槛

### 2.2 技术栈

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端 | 原生 HTML5 / CSS3 / JavaScript | 零依赖，SOLO 生成效率高，加载快 |
| 样式 | CSS 变量 + CSS Grid/Flexbox | 网格布局天然适合图标网格，响应式 |
| 语音 | Web Speech API (SpeechSynthesis) | 浏览器原生，支持中文语音播报 |
| 存储 | localStorage | 无需后端，本地持久化用户数据 |
| 图标 | Emoji + 内联 SVG | 零外部依赖，视觉直观 |
| 动画 | CSS transitions + keyframes | 轻量级交互动画 |
| PWA | Service Worker + Web Manifest | 离线可用，可添加到主屏幕 |
| 部署 | GitHub Pages / Vercel | 免费静态托管 |

### 2.3 项目结构

```
heart-bridge/
├── index.html          # 主页面（SPA 入口）
├── css/
│   └── style.css       # 全局样式
```

#### A3. 常用表达置顶

- 记录每条表达的使用次数（localStorage）
- 按使用频率排序，最常用的排在最前

### 3.3 模块 B：情绪选择器

#### B1. 情绪选择页面

- **功能描述**：当心智障碍员工无法准确描述自己的感受时，通过选择情绪图标来表达
- **展示内容**：5 种基础情绪，大图标 + 文字标签

| 情绪 | 图标 | 标签 | 颜色 |
|------|------|------|------|
| 紧张/不安 | 😰 | 我有点紧张 | 暖橙 |
| 困惑/不明白 | 😕 | 我不太懂 | 天蓝 |
| 沮丧/难过 | 😢 | 我有点难过 | 薰衣草 |
| 开心/满足 | 😊 | 我很开心 | 薄荷绿 |
| 疲惫/困倦 | 😴 | 我有点累 | 浅灰蓝 |

#### B2. 表达推荐

- 选择情绪后，展示 2-3 条推荐表达语句
- 每条语句配有朗读按钮
- 用户可选择最符合自己感受的语句

**情绪→表达映射表**：

| 情绪 | 推荐表达 1 | 推荐表达 2 |
|------|-----------|-----------|
| 紧张/不安 | 我有点紧张，可以慢慢来吗？ | 这个任务让我有点不安，能陪我一起做吗？ |
| 困惑/不明白 | 我不太明白，能再说一遍吗？ | 可以用图片告诉我怎么做吗？ |
| 沮丧/难过 | 我做了好几次都没做好，有点难过。 | 可以换一个任务让我试试吗？ |
| 开心/满足 | 我今天很开心！ | 我完成了任务，很有成就感！ |
| 疲惫/困倦 | 我有点累了，能休息一下吗？ | 我需要喝口水。 |

#### B3. 管理者沟通建议

- 选择情绪后，同时展示给管理者的沟通建议
- 建议包含：可能的原因分析 + 具体的应对策略
- 管理者可通过切换视图查看

**情绪→管理者建议映射表**：

| 情绪 | 管理者建议 |
|------|-----------|
| 紧张/不安 | 员工可能感到压力过大。建议：①分解任务为更小步骤 ②给予更多正面鼓励 ③允许短暂休息 ④避免催促 |
| 困惑/不明白 | 员工可能没有理解指令。建议：①使用更简单的语言 ②配合图片或示范 ③一次只说一件事 ④确认对方理解后再继续 |
| 沮丧/难过 | 员工可能遇到困难或感到挫败。建议：①肯定其努力 ②帮助分析具体卡在哪里 ③适当降低任务难度 ④给予更多正面反馈 |
| 开心/满足 | 员工状态良好。建议：①及时表扬和肯定 ②可以适当增加挑战 ③鼓励社交互动 ④记录积极表现 |
| 疲惫/困倦 | 员工可能体力不支。建议：①安排休息时间 ②检查工作强度是否合适 ③提供饮水和零食 ④考虑调整工作节奏 |

### 3.4 模块 C：沟通模板库

#### C1. 模板列表

- 以卡片形式展示可用的沟通模板
- 每个模板包含：图标、名称、简短说明

#### C2. 预设模板清单（MVP 阶段）

**模板一：汇报工作完成**

- 图标：📋
- 字段：
  - 任务名称（下拉选择）：理货 / 清洁 / 准备材料 / 收拾桌面 / 其他
  - 完成情况（下拉选择）：全部完成 / 大部分完成 / 需要帮助
- 输出示例："我已经完成了理货，全部完成，请检查。"

**模板二：请假申请**

- 图标：📅
- 字段：
  - 请假类型（下拉选择）：病假 / 事假 / 半天假
  - 原因（下拉选择）：身体不舒服 / 家里有事 / 去医院 / 其他
- 输出示例："我想请病假，因为身体不舒服，请问可以吗？"

**模板三：请求帮助**

- 图标：🙋
- 字段：
  - 需要帮助的事情（下拉选择）：不明白任务 / 找不到工具 / 做错了 / 需要人帮忙
- 输出示例："我不太明白这个任务，能帮帮我吗？"

**模板四：反馈问题**

- 图标：⚠️
- 字段：
  - 问题类型（下拉选择）：材料不够 / 工具坏了 / 东西放错了 / 其他
- 输出示例："这里的材料不够了，需要补充。"

#### C3. 填空交互设计

- **关键原则**：用下拉选择代替文字输入，心智障碍群体可能不擅长打字
- 每个模板不超过 3 个字段
- 选择后实时预览生成的完整句子
- 支持一键朗读和复制

### 3.5 模块 D：历史与收藏

#### D1. 使用历史

- 记录每次使用的表达：时间、分类、表达内容
- 按时间倒序展示，最多保留最近 50 条
- 支持快速重新使用（点击历史记录直接朗读）

#### D2. 收藏功能

- 长按或点击收藏按钮收藏常用表达
- 收藏的表达在首页「我的收藏」入口查看
- 收藏数据存储在 localStorage

#### D3. 使用统计

- 各分类使用频率（简易柱状图）
- 最常用的 5 条表达
- 本周使用次数

## 六、UI/UX 设计规范

### 6.1 设计原则

1. **极简交互**：每屏信息量极少，避免选择困难
2. **图标优先**：用大图标辅助文字理解，减少阅读依赖
3. **选择代替输入**：所有输入都用下拉选择，零打字
4. **语音优先**：所有表达都支持一键朗读
5. **温暖安全**：柔和暖色调，传递安全感和亲切感
6. **大触摸目标**：按钮最小 72px，间距充足

## 七、非功能需求

### 7.1 性能要求

- 首屏加载时间 < 2 秒（3G 网络）
- 页面切换流畅（60fps）
- 语音播报延迟 < 0.5 秒
- 复制到剪贴板即时完成

### 7.2 兼容性

- 支持主流移动浏览器：Chrome 80+、Safari 14+、微信内置浏览器
- 适配屏幕尺寸：375px - 768px
- Web Speech API 降级方案：不支持语音时显示文字提示
- Web Share API 降级方案：不支持分享时隐藏分享按钮

### 7.3 可访问性

- 所有交互元素支持键盘操作
- 足够的颜色对比度（WCAG AA）
- 支持系统字体缩放
- Emoji 图标配有文字标签

### 7.4 离线支持

- Service Worker 缓存所有静态资源
- 表达库数据内嵌，离线可用
- 历史记录和收藏本地存储

### 7.5 隐私安全

- 所有数据存储在本地（localStorage）
- 不收集、不上传任何个人信息
- 不需要注册或登录

## 十一、与「步步通」的差异化定位

| 维度 | 步步通 | 心语桥 |
|------|--------|--------|
| 核心场景 | 执行多步工作流程 | 日常沟通表达 |
| 交互模式 | 线性步骤推进 | 网格选择导航 |
| 解决痛点 | 忘记步骤、顺序混乱 | 表达困难、情绪无法传达 |
| AI 利用度 | 低（规则驱动） | 中（情绪→表达映射） |
| 目标感 | 提升独立工作能力 | 提升沟通能力和社交融入 |
| 互补性 | 可配合使用 | 可配合使用 |

两个产品可以**配合使用**：心智障碍员工在工作中同时使用「步步通」执行任务流程、使用「心语桥」进行沟通表达，形成完整的职场支持体系。
